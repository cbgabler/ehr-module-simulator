Available for Windows 10+ and macOS 10.15+ · Runs fully offline · No account required to install

The Problem

Nursing students need practice with EHR workflows before clinical rotations, but real EHR systems are inaccessible for training and too risky for trial-and-error learning.

Our Solution

A safe, offline desktop app that simulates real patient scenarios. Students titrate medications, watch vitals respond in real time, and document their reasoning — building clinical judgment before touching a real chart.

Who It's For

Built in partnership with Oregon Health & Science University for nursing faculty and students in clinical learning labs. Instructors author scenarios; students practice and get graded.

Key Features

Everything students and instructors need to practice clinical reasoning

01

Real-Time Patient Simulation

Patient vitals respond dynamically to your medication changes. A convergence-based pharmacokinetic model produces realistic vital sign curves — titrate an IV drip and watch blood pressure respond naturally over time.

02

Interactive Vitals Trend Graph

Track heart rate, blood pressure, SpO2, respiratory rate, and temperature on a live SVG chart. Hover for exact values, click the legend to filter, and watch how your clinical decisions affect outcomes tick by tick.

03

Instructor Scenario Authoring

Build custom patient scenarios with a guided form: set demographics, initial vitals, medication lists, provider orders, learning objectives, simulation parameters, and custom documentation tabs — no code required.

04

Session Summaries & PDF Export

Every action is logged automatically. When a session ends, students get a detailed timeline of their decisions — medication adjustments, pauses, completion reason — downloadable as a formatted PDF for submission or review.

05

Quizzes & Assessment

Instructors can create multiple-choice quizzes tied to learning objectives, assign them to students, and review scored submissions. Students complete quizzes in-app and see their results immediately.

Get Started

Up and running in under 2 minutes

1

Download

Grab the latest installer from the GitHub Releases page. Choose the .exe for Windows or .dmg for macOS.

2

Install

Run the installer. On Windows, you may need to click "More info" then "Run anyway" on the SmartScreen prompt. The app launches automatically when finished.

Sign In

Use the built-in demo accounts to get started, or register a new one. Two default accounts are created on first launch — one instructor, one student.

Frequently Asked Questions

Common questions about how the simulator works

What is a "tick"?

A tick is one simulation step. Each tick, the engine updates all vital signs: it applies baseline drift, calculates medication effects, clamps vitals to safe ranges, and checks whether target goals have been met. By default, a tick occurs every 5 seconds, but instructors can configure this from 1 to 60 seconds when authoring a scenario.

What is the difference between Training and Realistic mode?

Training mode (default) applies medication effects directly each tick, so vitals respond quickly and stabilize in roughly 5 ticks. This keeps simulations short and interactive for learning.

Realistic mode uses an exponential convergence model where vitals gradually approach a target shift over many ticks, producing pharmacokinetically accurate response curves. This takes longer to stabilize but mirrors how real medications behave in a patient. You can choose the mode when starting a simulation from the scenario details screen.

How does the simulation decide when I've "passed"?

If the scenario has target vital sign ranges configured, the simulation tracks how many consecutive ticks your patient's vitals stay within those ranges. Once the required number of consecutive ticks is reached (typically 2–3), the session auto-completes with a "Target Met" status. If no targets are configured, the simulation runs until you manually end it.

What is "baseline drift"?

Baseline drift is the natural change in vital signs that occurs each tick even without any medication adjustments. For example, a hypertension scenario might have blood pressure gradually rising over time, requiring the student to intervene with medications. Drift values are set by the scenario author.

What can instructors do that students cannot?

Instructors can create, edit, duplicate, and delete scenarios. They can also create and manage quizzes, assign quizzes to specific students, and view all user accounts. Students can start simulations, take quizzes, view their session summaries, and import/export scenario files.

What are the default login credentials?

Two accounts are created on first launch: instructor1 and student1, both with password password123. These are shared demo accounts — for real use, create individual accounts through the registration screen.

Does the app require an internet connection?

No. The app runs entirely offline. All data (scenarios, user accounts, quiz submissions, session summaries) is stored locally in a SQLite database on your computer.

Where is my data stored?

On Windows: C:\Users\<YourName>\AppData\Roaming\ehr-module-simulator\. On macOS: ~/Library/Application Support/ehr-module-simulator/. Uninstalling the app does not delete this folder — remove it manually if you want to clear all data.

Can I share scenarios with other users?

Yes. Use the Export button on the Home page to save selected scenarios as a portable .db file. Another user can then use the Import button to load those scenarios into their installation. Duplicate scenarios (same name and definition) are automatically skipped during import.

What are custom tabs?

Custom tabs are instructor-defined documentation sections that appear alongside the standard simulation tabs (Vitals, Medications, Orders, etc.). They can include fields like urine output, wound assessment, or any structured data the instructor wants students to document. Custom tab values do not affect the simulation engine — they serve as a documentation workspace.

What keyboard shortcuts are available?

Space — pause or resume the simulation. 1–9 — switch between tabs (Vitals, Medications, Orders, etc.). N — focus the notes text area. Escape — end the simulation (opens confirmation). Shortcuts are disabled while typing in an input field.

I see a blank white screen when the app launches. What do I do?

This occasionally happens on the very first launch while the database initializes. Close the app and reopen it — the issue should not recur.
